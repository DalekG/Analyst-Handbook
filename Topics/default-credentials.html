Use of Default or Weak Credentials

Overview

The use of default or weak credentials remains one of the most common entry points for adversaries. As of 2025, automated "spraying" bots continuously target known manufacturer defaults and common password patterns. Identifying these vulnerabilities before or during an attack is vital to preventing unauthorized access and lateral movement.

Attack Description

Adversaries leverage credential-based attacks such as:

  • Default Credential Testing: Using "admin/admin" or "root/password" on new hardware.
  • Password Spraying: Testing common passwords (e.g., "Company2025!") against many accounts.
  • Brute Force: Systematically attempting to guess passwords on exposed services.
  • IoT Exploitation: Targeting embedded devices that lack robust password enforcement.

Investigation Workflow

Identify Where Detected

What system/service is involved?

  • Critical Server: (DC, DB, Web) Immediate remediation required.
  • Network Device: (Router, Switch, Firewall) High priority; risk of traffic interception.
  • Application: Check the data sensitivity and exposure level.
  • IoT/Embedded Device: Often overlooked; can be used as a persistent foothold.

How was it detected?

  • Successful login with a known default credential (via SIEM/EDR).
  • Vulnerability scan finding (e.g., Nessus, Qualys).
  • Internal password audit or "cracking" exercise.
  • Failed attack attempt that revealed the use of a default credential in the logs.

Assess Exposure and Risk

  • Is the system internet-facing? If Yes, this is a CRITICAL finding requiring immediate change.
  • What level of access is provided?
    • Administrative/Root access: Critical risk.
    • User-level access: Serious risk.
    • Read-only access: Moderate concern; still requires remediation.
  • Historical Context: How long has this been in use? Review logs for unauthorized access over the last 90 days.

Has the Credential Been Abused?

Review Authentication Logs:

  • Identify unusual login times, sources (IPs), or geolocations.
  • Look for a pattern of multiple failed attempts followed by a single success.

Review System Activity:

  • Check for unauthorized configuration changes.
  • Analyze file or database access logs for data staging.
  • Look for unauthorized software or script installations.

Findings:

  • Yes (Signs of Abuse): Escalate immediately to the Account Compromise workflow. Assume the system is compromised.
  • No (No signs of abuse): Proceed to Immediate Remediation.

Immediate Remediation

  • Change the credential immediately to a unique, complex password or secret.
  • Force a logout of all active sessions associated with the account.
  • Review and harden the security configuration (e.g., disable unused services).
  • Document the specific default/weak credential found for reporting.
  • Check all other systems from the same vendor/model for identical defaults.
  • If the password was user-created, update the Password Policy to enforce complexity.

Tools & Queries

  • Vulnerability Scanners (Nessus, OpenVAS)
  • SIEM Search: EventCode=4624 (Logon) Filter by known default usernames.
  • Password Auditing Tools (Hashcat, John the Ripper - authorized use only)
  • Asset Inventory / CMDB (to identify similar hardware)

Response Workflow

  • Perform a broader environment scan for other instances of default credentials.
  • Update system hardening standards and gold images.
  • Issue a security awareness notification if the weak password was user-created.
  • Verify remediation with a follow-up vulnerability scan.
  • Request Senior Analyst review of the incident closure.
