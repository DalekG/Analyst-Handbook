Account Compromise

Overview

Account compromise occurs when an unauthorized individual gains access to a legitimate user's credentials. This is a high-priority event as it allows adversaries to bypass the perimeter and act with the authority of the compromised user, often leading to data theft, lateral movement, or financial fraud.

Attack Description

Compromise typically results from phishing, credential stuffing, password spraying, or session hijacking. Once inside, attackers often:

  • Modify mailbox rules to hide communication
  • Enroll unauthorized MFA devices for persistence
  • Access sensitive documents in SharePoint or OneDrive
  • Use the account to launch internal phishing attacks against other employees

Investigation Workflow

Identify Compromise Indicators

What triggered the alert?

  • Unusual login location (Geo-proximity) or anomalous time
  • Impossible Travel: Logins from distant locations in an impossible timeframe
  • Successful login following a known Credential Dump event
  • Successful login following a Brute Force or Password Spraying attempt
  • Suspicious activity post-authentication (e.g., mass file downloads)
  • Direct report from the user regarding unauthorized access or locked account

Verify Compromise

Check account activity:

  • Review recent login times and IP sources
  • Analyze failed login attempts (pattern of failure before success)
  • Check for recent password changes or self-service resets
  • MFA Status: Review changes to MFA methods or high volumes of "Denied" prompts
  • Review mailbox rules (e.g., rules to move items to "Deleted Items" or external forwarding)
  • Review file access patterns (unusual access to PII or Intellectual Property)
  • Check for any privileged actions performed (e.g., global admin changes)

Contact Account Owner:

  • Verify recent activity was legitimate via phone or secure IM
  • Confirm their physical location matches the logs
  • Inquire about unexpected password reset or MFA notification emails
  • Ask if they have shared credentials or entered them into any suspicious portals

Is the account actually compromised?

  • Yes or Likely:
    • Proceed immediately to Response Workflow
  • Unclear:
    • Request Senior Analyst review
    • Enable enhanced auditing/monitoring on the account
    • Prepare for rapid isolation if confirmation is received
  • No (False Positive):
    • Document the reason (e.g., Authorized VPN use, user on travel)
    • Tune the alert logic if necessary
    • Close the alert

Execute Immediate Response

  • Disable the account immediately in AD / Azure AD
  • Terminate all active sessions (Revoke Refresh Tokens)
  • Reset the password (Note: Do not send the new password to the compromised email)
  • Review and remove persistence mechanisms:
    • Email forwarding rules and mailbox delegations
    • Unauthorized OAuth app tokens and permissions
    • Saved passwords in browser or cloud vaults
    • Newly enrolled MFA devices

Assess Damage and Scope

What data was accessed?

  • Sensitive/Classified information or Intellectual Property
  • PII, financial data, or customer information (GDPR/HIPAA impact)

What systems were accessed?

  • Document every system/SaaS app logged into during the compromise window
  • Check for lateral movement to other internal assets
  • Verify no additional accounts were compromised by this user

What actions were taken?

  • Modifications to security settings or logging configurations
  • Installation of unauthorized software or scripts
  • Creation of additional persistence accounts
  • Evidence of data exfiltration (outbound traffic volume)

Tools & Queries

  • Azure AD Sign-in Logs / Risky Users Report
  • Microsoft 365 Unified Audit Log (UAL)
  • SIEM Search: EventCode=4624 (Logon) and EventCode=4723/4724 (Password Change)
  • MFA Provider Dashboard (Okta, Duo, Azure MFA)

Response Workflow

  • Escalate to IR with a full impact assessment report
  • Begin official Incident Response procedures and evidence preservation
  • Notify relevant stakeholders (Legal, HR, Privacy) per the IR plan
  • Initiate a threat hunt for similar indicators across the environment
