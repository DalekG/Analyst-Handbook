Large Data Transfers

Overview

Large data transfers (Data Heaviness) involve significant volumes of data moving across network boundaries. This workflow helps distinguish between legitimate business operations (backups, updates) and potential Data Exfiltration or Staging.

Attack Description

Adversaries often aggregate data into large archives before moving them to external repositories. Detection depends on monitoring for deviations from established traffic baselines for specific users or hosts.

Investigation Workflow

Characterize the Transfer

Direction and Volume

  • Outbound (Upload): High concern for exfiltration. Compare against the 7-day or 30-day average.
  • Inbound (Download): Large software updates or patches. Potential for "ingress" of large malicious toolsets.
  • Internal (Lateral): Possible data staging; moving data from multiple hosts to a central collection point.

Identify the Destination

  • Sanctioned Cloud: (OneDrive, SharePoint, AWS) Verify if the user/system is authorized for this volume.
  • Unsanctioned Site: (Mega.nz, WeTransfer, Personal Dropbox) High suspicion; often used for exfiltration.
  • Unknown IP/Geo: Check IP reputation and geolocation for high-risk or sanctioned regions.

Identify the Source and User

  • System Type: A Database server uploading 50GB is more critical than a Creative Dept workstation uploading video files.
  • User Context: Is the user currently on shift? Does this align with their job role?
  • Process Activity: Check EDR logs for file compression tools (7z.exe, WinRAR.exe, tar) running prior to the transfer.

Determine Legitimacy

  • Scheduled Tasks: Check for active backup windows, cloud syncs, or database replications.
  • Approved Projects: Contact the user or manager to verify if a large data migration was planned.
  • Anomalous Activity: If no business justification is found and the destination is suspicious, proceed to response.

Response Actions

  • Containment: Kill active network sessions or block the destination URL/IP at the proxy/firewall.
  • Evidence Preservation: Capture netflow data and proxy logs. If internal, capture file-access audit logs.
  • Escalation: If the data is suspected to be PII, PHI, or IP, trigger the Data Exfiltration Playbook and notify Legal/Compliance.

Tools & Queries

  • NetFlow Analysis (Top Talkers by volume)
  • Proxy/Web Gateway Logs (Look for PUT or POST methods)
  • EDR (Look for large file reads or compression processes)
  • DLP (Data Loss Prevention) Console
