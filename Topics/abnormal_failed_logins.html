Abnormal Failed Account Logins

Home | Next Topic

Overview

This section covers detection, analysis, and remediation of abnormal failed account logins. These events can indicate brute-force attempts, credential stuffing, or misconfigured applications.

Attack Description

  • Goal: Gain unauthorized access through repeated failed logins.
  • Common Sources: External IPs, automated scripts, compromised devices.
  • Indicators: High login failure rate, unusual user accounts, unusual user activity (same user across multiple devices).

Detection Workflow

  1. Identify patterns in authentication logs (e.g., multiple failures within a short window).
  2. Correlate IPs, usernames, and timestamps across systems.
  3. Check for known brute-force tools or login automation signatures.
  4. Determine whether the failures are from valid users or external sources.

Analysis Workflow

This workflow guides the analyst through triaging abnormal failed account logins based on context and log source.

  • Local Logon Type?
    • Yes: Conduct the following actions:
      • 1. Check for repeated failed logins from the same workstation.
      • 2. Contact system owner, AO help-desk, or section
      • 3. Verify if the user recently changed passwords or had lockouts.
      • 4. Identify reason for failed logins.
    • No:
      • From a trusted network?
        • Yes:
          • 1. Contact network owner, and identify reason for activity
          • 2. Review system and user activity logs for internal threat indicators.
          • 3. Validate external network ability/reason for access.
        • No:
          • Did this happen across multiple machines?
            • Yes:
            • 1. Determine if malicious application caused failures.
            • 2. Determine if network or application authentication caused failures.
            • 3. Determine if successful login occurred.
          • No:
            • Is this an Incident?
              • Yes:
              • 1. Gather artifacts (logs, affected hosts, malicious application, source IP address)
              • 2. Prepare cyber 9-line
              • 3. Senior CND analyst review
          • No:
            • 1. Senior CND analyst review.
            • 2. Determine false positives and eliminate them.

Response Workflow

  1. Senior CND analyst review.
  2. Escalate to SSS for account validation and review.
  3. Block malicious IP addresses.
  4. Force password resets if credential compromise is suspected.
  5. Gather artifacts (logs, affected accounts, affected computers, source IP addr, etc.)
