Abnormal Failed Account Logins

Home | Next Topic

Overview

This section covers detection, analysis, and remediation of abnormal failed account logins. These events can indicate brute-force attempts, credential stuffing, or misconfigured applications.

Attack Description

  • Goal: Gain unauthorized access through repeated failed logins.
  • Common Sources: External IPs, automated scripts, compromised devices.
  • Indicators: High login failure rate, unusual user accounts, geolocation anomalies.

Detection Workflow

  1. Identify patterns in authentication logs (e.g., multiple failures within a short window).
  2. Correlate IPs, usernames, and timestamps across systems.
  3. Check for known brute-force tools or login automation signatures.
  4. Determine whether the failures are from valid users or external sources.

Analysis Workflow

This workflow guides the analyst through triaging abnormal failed account logins based on context and log source.

  • Local Logon Type?
    • Yes: Conduct the following actions:
      • Check for repeated failed logins from the same workstation.
      • Verify if the user recently changed passwords or had lockouts.
      • Correlate with help desk or self-service reset activity.
    • No:
      • From a trusted network?
        • Yes:
          • Review system and user activity logs for internal threat indicators.
          • Validate MFA logs for sign-in success or denial.
        • No:
          • Check geolocation and ASN reputation of the source IP.
          • Query threat intel feeds for malicious indicators.
          • Proceed to escalation and incident creation if verified malicious.

Tools & Queries


index=auth sourcetype=windows:security EventCode=4625
| stats count by user, src_ip
| where count > 10
      

SigninLogs
| where ResultType != 0
| summarize FailedLogins=count() by UserPrincipalName, IPAddress
| where FailedLogins > 5

Response Workflow

  1. Validate the affected accounts and notify users if necessary.
  2. Block or rate-limit malicious IP addresses.
  3. Force password resets if credential compromise is suspected.
  4. Review MFA enforcement and password policy configurations.
