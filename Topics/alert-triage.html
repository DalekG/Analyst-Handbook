Alert Triage

Overview

Alert triage is the critical first phase of the incident response lifecycle. This process ensures that security events are prioritized, validated, and categorized correctly to ensure SOC resources are focused on genuine threats while minimizing the noise from false positives.

Description

The triage process involves analyzing incoming alerts from various security platforms (SIEM, EDR, IDS) to determine their validity. Effective triage prevents "alert fatigue" and ensures that indicators of compromise (IOCs) are contained before they can escalate into full-scale breaches.

What to Look For

  • Known malicious hashes, IPs, or domains (IOCs)
  • Deviations from established user or system baselines
  • Multiple related alerts firing against a single asset
  • Unusual administrative activity or privilege escalation
  • Signatures matching known adversary tactics (MITRE ATT&CK)

Investigation Workflow

Alert Triage

Contain IOC?

  • Yes
    • Start a hunt playbook
    • Dig in to find other assets with the same IOC
    • Pivot to broader environmental scanning
  • No
    • Proceed to next step

Is this alert from an inside resource?

  • Yes
    • Identify that there isn't a malicious application present
    • Identify there is no malicious network traffic to or from the resource/asset
    • Determine whether this is normal or abnormal traffic/event based on baseline
  • No
    • Proceed to next step

Is this a real problem? (True Positive)

  • Yes
    • Request Senior CND analyst review
    • Develop a Cyber 9-Line report
    • Escalate immediately to Incident Response (IR)
  • No
    • Proceed to next step

Do we want to be alerted on this activity?

  • Yes
    • Request Senior CND analyst review
    • Determine if any tweaks need to be made to the logic of the rule
    • Document why the alert is valuable despite this specific instance
  • No
    • Request Senior CND analyst review
    • Determine what specific telemetry triggered the false positive
    • Tune the alert/correlation rule to prevent future recurrences

Tools & Queries

  • SIEM Dashboard (Elastic, Splunk, etc.)
  • EDR Console (CrowdStrike, SentinelOne, Carbon Black)
  • Threat Intelligence Feeds (VirusTotal, AlienVault)
  • Internal Asset Inventory / CMDB
  • Cyber 9-Line Reporting Template

Response Workflow

  • Verify the integrity of the alert source
  • Execute containment if IOCs are confirmed
  • Coordinate with Senior Analysts for rule tuning or escalation
  • Ensure all Cyber 9-Line details are accurate before IR handoff
  • Close out False Positives with detailed tuning notes
